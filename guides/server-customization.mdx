# Server Customization Guide

Transform your RYLM server from default to extraordinary! This guide covers everything you need to know about customizing your Hytale server.

## Server Settings

### Access Server Settings

1. Log in to your RYLM dashboard
2. Select your server
3. Click the **"Settings"** tab

### Basic Configuration

#### Server Properties

**MOTD (Message of the Day)**
Customize the message players see in the server list:

```
Welcome to my Epic Hytale Server!
Adventure Awaits in Orbis!
```

**Max Players**
Adjust player capacity (1-10 for free tier):
- Small server: 4-6 players
- Medium server: 6-8 players
- Large server: 8-10 players

**View Distance**
Control how far players can see:
- Low (6 chunks) - Better performance
- Medium (8 chunks) - Balanced
- High (12 chunks) - Best visuals

**Server Icon**
Upload a custom 64x64 PNG image to represent your server.

### Gameplay Settings

#### Game Rules

Access via **Settings â†’ Game Rules**:

```yaml
# PvP Settings
pvp: true
friendly-fire: false

# Difficulty
difficulty: normal
hardcore: false

# Time & Weather
do-daylight-cycle: true
do-weather-cycle: true

# Spawning
spawn-monsters: true
spawn-animals: true
spawn-npcs: true

# Block Settings
fire-spread: false
mob-griefing: false
```

#### World Settings

**Default World**
- Seed: Enter a custom seed or leave blank
- World Type: Normal, Flat, Custom
- Generate Structures: Enable/disable dungeons, villages, etc.

**Additional Worlds**
Create multiple worlds:

1. Go to **Settings â†’ Worlds**
2. Click **"Create New World"**
3. Configure world settings
4. Link portals between worlds

## Installing Plugins

### Browse the Plugin Library

1. Navigate to **Plugins â†’ Browse**
2. Search or filter by category:
   - **Economy** - Virtual currencies and shops
   - **Protection** - Land claiming and grief prevention
   - **Minigames** - PvP arenas, races, challenges
   - **Cosmetics** - Custom items and effects
   - **Utilities** - Quality of life improvements
   - **Admin Tools** - Server management
   - **Social** - Chat, parties, friends

### Popular Plugins for Beginners

#### Essential Protection

**LandClaim**
```
Features:
- Claim land to prevent griefing
- Allow/deny specific players
- Create trusted groups
- GUI-based management
```

**CoreProtect**
```
Features:
- Rollback griefing
- Inspect block changes
- Track player actions
- Restore deleted areas
```

#### Economy System

**HytaleEconomy**
```
Features:
- Virtual currency system
- Player-to-player trading
- Admin commands for balances
- Integration with shop plugins
```

**ChestShop**
```
Features:
- Create shops with chests
- Buy and sell items
- Automatic transactions
- Tax system
```

#### Quality of Life

**Essentials**
```
Features:
- Home and warp commands
- Teleportation
- Kits and items
- Nickname support
- And much more!
```

**QuickTeleport**
```
Features:
- /spawn, /home, /back commands
- Teleportation requests
- Cooldown management
- Permission-based access
```

### Installing a Plugin

1. Browse or search for a plugin
2. Click on the plugin card
3. Click **"Install"**
4. Wait for installation (10-30 seconds)
5. Restart your server
6. Plugin is now active!

### Configuring Plugins

After installation:

1. Go to **Plugins â†’ Installed**
2. Click the plugin name
3. Edit configuration in the web editor
4. Click **"Save"**
5. Restart server to apply changes

Example plugin config:

```yaml
# LandClaim Configuration
max-claims-per-player: 3
claim-blocks-per-hour: 100
allow-explosions-in-claims: false
allow-pvp-in-claims: false

default-permissions:
  build: false
  interact: false
  access-chests: false
```

## Appearance Customization

### Server Resource Packs

Add custom textures, models, and sounds:

1. Go to **Settings â†’ Resource Packs**
2. Upload your resource pack (.zip file)
3. Enable "Force Resource Pack" (optional)
4. Save changes

**Recommended Size**: Keep under 50MB for faster downloads

### Custom Messages

Personalize server messages:

**Join/Leave Messages**
```yaml
join-message: "&e{player} has entered the realm!"
leave-message: "&e{player} has departed..."
```

**Death Messages**
```yaml
custom-death-messages: true
death-message-format: "&c{player} {death_cause}"
```

**Chat Format**
```yaml
chat-format: "&7[{rank}] &f{player}: {message}"
```

## Permission Systems

### Setting Up Ranks

Create custom ranks with different permissions:

1. Go to **Settings â†’ Permissions**
2. Click **"Create Rank"**
3. Configure rank settings:

```yaml
Rank: VIP
Prefix: "[VIP]"
Color: "&6"
Permissions:
  - essentials.fly
  - landclaim.bonus.blocks
  - chat.colors
Priority: 10
```

### Common Permission Groups

**Default Player**
```
- Basic gameplay
- Chat access
- /spawn, /help
```

**VIP**
```
- All default permissions
- /fly
- Extra land claims
- Colored chat
- Cosmetic effects
```

**Moderator**
```
- All VIP permissions
- /kick, /mute
- /tp to players
- View reports
- Manage chat
```

**Admin**
```
- All permissions
- Server management
- Plugin configuration
- World editing
```

### Assigning Ranks

```bash
# Via console
/rank set <player> <rank>

# Examples
/rank set Steve VIP
/rank set Alex Moderator
```

## Advanced Customization

### Server Scripts

Create custom scripts for automation:

1. Go to **Settings â†’ Scripts**
2. Click **"Create Script"**
3. Choose trigger:
   - Player Join
   - Player Leave
   - Server Start
   - Scheduled (cron)

Example welcome script:

```javascript
// Trigger: Player Join
function onPlayerJoin(player) {
  server.broadcast(`Welcome ${player.name}!`);
  player.sendMessage("Type /help for commands!");
  player.giveItem("wooden_sword", 1);
}
```

### Custom Commands

Create shortcuts for complex operations:

```yaml
# Settings â†’ Custom Commands
/kit starter:
  type: command-sequence
  commands:
    - give @p wooden_sword 1
    - give @p bread 16
    - give @p torch 32
  cooldown: 86400 # 24 hours

/pvp:
  type: teleport
  location: pvp-arena
  message: "Entering PvP arena!"
```

### Scheduled Events

Automate server tasks:

```yaml
# Settings â†’ Scheduled Tasks
daily-restart:
  schedule: "0 4 * * *" # 4 AM daily
  action: restart
  warning: true
  warning-times: [10, 5, 1] # minutes

auto-save:
  schedule: "*/30 * * * *" # Every 30 minutes
  action: save-all

announcement:
  schedule: "0 */2 * * *" # Every 2 hours
  action: broadcast
  message: "Remember to vote for the server!"
```

## Performance Optimization

### Optimize Server Settings

**View Distance**
Reduce for better performance:
```yaml
view-distance: 8  # Default
view-distance: 6  # Better performance
```

**Entity Management**
```yaml
max-entities-per-chunk: 50
remove-inactive-entities: true
entity-activation-range:
  animals: 32
  monsters: 48
  misc: 16
```

**Chunk Management**
```yaml
chunk-gc-period: 600
max-chunk-sends-per-tick: 5
```

### Plugin Optimization

- Only install plugins you actually use
- Disable unused plugin features
- Update plugins regularly
- Monitor plugin performance in dashboard

## Backup and Restore

### Automatic Backups

Enable automatic backups:

1. Go to **Settings â†’ Backups**
2. Enable **"Automatic Backups"**
3. Set frequency:
   - Every 6 hours
   - Daily
   - Weekly

**Note**: Free tier includes 3 backup slots

### Manual Backup

Create a backup anytime:

1. Go to **Backups** tab
2. Click **"Create Backup"**
3. Add a description
4. Wait for completion

### Restoring from Backup

1. Go to **Backups** tab
2. Find the backup you want
3. Click **"Restore"**
4. Confirm restoration
5. Server will restart automatically

## Security Best Practices

### Enable Whitelist

For private servers:

```bash
/whitelist on
/whitelist add <username>
```

### Set Up Login Security

1. Enable **"Online Mode"** for authentication
2. Use **"Login Plugin"** for password protection
3. Enable **"2FA"** for operators

### Protect Against Griefing

1. Install **CoreProtect** for rollback
2. Install **LandClaim** for protection
3. Enable **Container Protection**
4. Disable **TNT and Fire Spread**

### Regular Monitoring

- Check logs daily for suspicious activity
- Review plugin reports
- Monitor player behavior
- Update plugins and server software
